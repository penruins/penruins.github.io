    关于 Nacos 配置更新后避免 HikariCP 连接断开的问题#

    解决方案#

    使用 @RefreshScope(proxyMode = ScopedProxyMode.NO) 排除数据源#

    @Configuration
@RefreshScope(proxyMode = ScopedProxyMode.NO)
public class DataSourceConfig {
    // 数据源配置
}

    自定义 RefreshScope 排除数据源相关配置#

    @Bean
public RefreshScope refreshScope() {
    RefreshScope scope = new RefreshScope();
    scope.setExcludeNames(new String[]{"dataSource", "hikariDataSource"});
    return scope;
}

    在 bootstrap.yml 中排除特定配置的刷新#

    spring:
  cloud:
    nacos:
      config:
        refresh-enabled: true
        refreshable-data-sources: none # 不刷新数据源配置

    使用 @RefreshScope 但不刷新数据源#

    @Bean
@ConfigurationProperties(prefix = "spring.datasource.hikari")
@RefreshScope(proxyMode = ScopedProxyMode.NO)
public HikariDataSource dataSource() {
    return DataSourceBuilder.create().type(HikariDataSource.class).build();
}

    最佳实践#

    1. 分离配置：将数据库配置和其他频繁变动的配置分开，放在不同的配置文件中
    2. 版本控制：对数据库配置的修改应该通过版本控制和应用重启来完成，而不是热更新
    3. 监控：即使解决了这个问题，也应该监控数据库连接池状态

    注意事项#

    • 这些解决方案可能会因 Spring Boot/Cloud 版本不同而有所差异
    • 在生产环境实施前应在测试环境充分验证
    • 如果确实需要动态修改数据库配置，建议使用更优雅的方式如多数据源切换
