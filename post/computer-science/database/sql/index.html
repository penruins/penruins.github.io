    177 第N高的薪水 #

    • 177 第N高的薪水
    create function getNthHighestSalary(N int) returns int
begin
    declare M int;
    set M = N - 1;
    return (select distinct salary
            from Employee
            order by salary desc
            limit M, 1);
end

    select S1.score,
       (select count(distinct S2.score)
        from Scores S2
        where S2.score >= S1.score) as 'rank'
from Scores S1
order by S1.score desc;

    select S.score,
       count(distinct T.score) as 'rank'
from Scores S
         inner join Scores T on S.score <= T.score
group by S.id,
         S.socre
order by S.score desc;

    数据库表自身进行内连接的结果是什么 #

    数据库表自身进行内连接（Self-Join）的结果是将表中的每一行与同一表中的其他行进行匹配，基于指定的连接条件。如果连接条件满足，则返回匹配的行组合。

    具体说明：

    1. 连接条件：通常使用主键、外键或其他相关列作为连接条件。
    2. 结果：返回满足条件的行组合，未匹配的行将被排除。

    示例：

    假设有一个表 Employees：

    EmployeeID Name ManagerID
    1 Alice 3
    2 Bob 3
    3 Charlie NULL
    4 David 2

    执行以下自连接查询：

    SELECT e1.Name AS Employee, e2.Name AS Manager
FROM Employees e1
         INNER JOIN Employees e2 ON e1.ManagerID = e2.EmployeeID;

    结果：

    Employee Manager
    Alice Charlie
    Bob Charlie
    David Bob

    解释：

    • Alice 和 Bob 的 ManagerID 为 3，与 Charlie 的 EmployeeID 匹配。
    • David 的 ManagerID 为 2，与 Bob 的 EmployeeID 匹配。
    • Charlie 没有 ManagerID，因此不参与匹配。

    总结：

    自连接的结果是表中满足连接条件的行组合，常用于处理层次结构或递归关系的数据。

    方法 1：分组并提取最小值 # 

    select player_id,

    方法 2：窗口函数 #

    rank() # 

    select X.player_id,
       X.event_date as first_login
from (select A.player_id,
             A.event_date rank() over (
            partition by A.player_id
            group by A.event_date
        ) as rnk
      from Activity A) X
where X.rnk = 1;

    first_value() # 

    --- 必须使用 distinct
SELECT DISTINCT A.player_id,
                FIRST_VALUE(A.event_date) OVER (
                    PARTITION BY
                        A.player_id
                    ORDER BY
                        A.event_date
                    ) AS first_login
FROM Activity A;

    在sql中，from后面用逗号写了多张表，这样属于什么连接 #

    在 SQL 中，如果在 FROM 子句中使用逗号分隔多张表，例如：

    FROM table1, table2

    这属于 隐式连接（Implicit Join），也称为 笛卡尔连接（Cartesian Join）或 交叉连接（Cross Join）。它的行为是将两张表的所有行进行组合，生成一个笛卡尔积。

    隐式连接的特点：

    1. 没有明确的连接条件：

      • 如果没有在 WHERE 子句中指定连接条件，结果将是两张表的笛卡尔积。
      • 例如，table1 有 3 行，table2 有 4 行，结果将返回 (3 \times 4 = 12) 行。

    2. 连接条件需要在 WHERE 子句中指定：

      • 如果需要特定的连接逻辑（如内连接），必须在 WHERE 子句中明确写出连接条件。
      • 例如： 
        FROM table1, table2
WHERE table1.id = table2.table1_id;

    3. 可读性较差：

      • 隐式连接的语法不够直观，尤其是在涉及多张表或复杂连接条件时，容易出错或难以维护。

    示例：

    假设有两张表：

    table1:

    id name
    1 Alice
    2 Bob

    table2:

    id table1_id value
    1 1 100
    2 2 200
    3 1 300
    1. 隐式连接（无 WHERE 条件）：
    SELECT *
FROM table1,
     table2;

    结果（笛卡尔积）：

    id name id table1_id value
    1 Alice 1 1 100
    1 Alice 2 2 200
    1 Alice 3 1 300
    2 Bob 1 1 100
    2 Bob 2 2 200
    2 Bob 3 1 300
    1. 隐式连接（带 WHERE 条件）：
    SELECT *
FROM table1,
     table2
WHERE table1.id = table2.table1_id;

    结果（内连接效果）：

    id name id table1_id value
    1 Alice 1 1 100
    1 Alice 3 1 300
    2 Bob 2 2 200

    显式连接 vs 隐式连接：

    • 显式连接（推荐）：

      • 使用 JOIN 关键字（如 INNER JOIN、LEFT JOIN 等）。
      • 连接条件写在 ON 子句中。
      • 语法清晰，易于维护。
      • 示例： 
        SELECT *
FROM table1
INNER JOIN table2 ON table1.id = table2.table1_id;

    • 隐式连接（不推荐）：

      • 使用逗号分隔表。
      • 连接条件写在 WHERE 子句中。
      • 可读性差，容易出错。

    总结：

    在 FROM 子句中使用逗号分隔多张表属于 隐式连接。如果没有指定 WHERE 条件，结果将是笛卡尔积；如果指定了 WHERE 条件，可以实现类似内连接的效果。但出于可读性和可维护性的考虑，建议使用显式连接（JOIN）。
