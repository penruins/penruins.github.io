小论文

一、感想 听完了第一节课，“合作”的重要性又一次被提及。合作时刻都存在着，无论是在生活中、学习中亦或是工作中。 如果我们想要寻求自身的发展或是获得快乐的生活，都需要合作。首先在这个分工高度细化的社会，以前能够一个人完成的事情越来越少，无论是开发一个计算机软件，建造一幢建筑，运营一档节目，都是需要一个团队甚至是几个团队的支持与合作。我认为可以这么说，在现在的社会单独一个人什么都做不了，我们间接或直接地都在与他人合作。其次，根据社会心理学的相关理论和我自己的经验，人是社会性动物，我们天生有与他人发生联系的倾向。在与别人相处的过程中，我们能缓解孤独与焦虑，产生愉悦的心情，获得动力，积极地看待生活。如果我们想要追求上述两方面，我们必须学会合作。

我认为合作有两个主要的影响因素：相似的背景知识、同理心。

首先，两个人必须得有相似知识背景，交流与合作才有可能。一个编程能力不好的人是不能与他人很好地合作编写代码的，一个思维没有逻辑性的人往往不能与别人很好地讨论。 其次，合作的顺利进行还需要同理心，我们得站在别人的角度考虑别人的想法。就其本质，同理心是一种理性思维，我们要理性地认识人，这里的人是一个广泛意义的概念，是人的共同特征的集合。例如，A与B的合作过程中，A总是在工作中拖沓，不负责任地把任务推给B，B在A的面前并没有表现得很反感或有意见，双方就这样继续着合作。我们不再考虑后面会发生什么，仅从这一个片段中来作一个简单地分析。在A与B的合作中，如果A是一个不具备同理心的人，那么A会相信他得到的直接反馈（也就是他的行为没有什么不得当）；如果A是一个具备同理心的人，那么A虽然得到了正面反馈，但他会做进一步的理性思考。我们假设他当时是这样想的：我在与B的合作中占便宜，态度不积极，对B是不公平的。虽然B没有表现出异议，但人都是喜欢公平的，因此B只是没有表现出来，或许是为了不让我难堪。我要在A没有直接表现出反感之前做出改变。我认为同理心就是这样一个三段论：1（人具有某种特质）2（A是人）3（A具有某种特质）。从这个角度出发，同理心的培养就是真正认识广泛意义的人所具备的特征。从一般的人的角度来认识具体的人。我个人认为，在生活中我们很难培养这样的认识，我们倾向于与和我们相似的人相处，并且有群体极化的倾向，无论是在家中还是在学校里。我认为学习心理学的相关知识如社会心理学在这方面有很多帮助。

二、时间与精力的分配 从进入大学开始到现在，“迷茫”这个词一直萦绕在我的脑海中。挣脱了管束之后的我对自己的身份产生了质疑。”学习“——这个我被决定的任务开始不再那么坚定。在不确定与迷茫中，我毫无目的地过着在其他人描述中“精彩的大学生活“，大概这样的状态持续了一年的时间，空虚与痛苦最后迫使着我开始改变。我开始思考，在阅读中思考。这个过程可以用两个词语来概括：批判与启蒙。批判与质疑之前的想法，用新的思想启蒙自我。这样的状态也持续了一年的时间。大三上学期，我开始把更多的时间用于课程的学习。静心学习与努力是我最后思考的结果，但这一次的学习是我主动选择的。也就是我到了黑格尔所说的合题的阶段。从一开始被动接受，到抗拒，最后回到一开始的选择。当然每一个人思考的结果是不一样的，只是我的过程正好是合题。 总结来看，我的大学的时间与精力的分配相比起那些一开始就目标明确的同学而言显得很糟糕，我的很多时间被”迷茫“所充满，很多时间又被“寻找方向”所充满，投入学习的时间很有限。但还好现在比以前好了一些。改变不是一蹴而就，需要一步一步往前走。

三、形成的能力 首先是在我寻找方向的过程中养成了思考和阅读的习惯，我感受到在网络信息充斥的现实世界中，阅读与思考能让我更加理性与宁静的看待自己的生活以及自己的环境 大学中的认识的同学在性格和认识方面都与自己有很大的不同，在与他们的接触中，慢慢学会与他人融洽相处的能力 学习从一件被动的事情转变为一件主动的事情 积极提问的习惯（薛老师上课时提到在工作中要有自己寻找答案并解决问题的能力，切忌在学校里一样。上课时听到老师这么说心里很矛盾，希望能在课程中慢慢体会）

  6. 以下哪些是软件项目的特征？ABCD
    A 设计流程创新
    B 风险难以预见及控制
    C 进度及质量难以估算
    D 满足不同人群的需求

  7. 试分析一个成功项目必须满足的基本要素。
    在工程允许的范围内满足成本，进度和客户满意的产品质量。

  8. 什么是项目的软件过程？它与开发流程及管理过程有什么区别和联系？
    项目的软件过程：用于从事软件开发及维护的全部技术，方法，活动，工具，以及他们之间的相互变换统称为软件过程。
    开发流程：一种活动序列，如同流水生产线上的生产过程
    管理过程：对过程进行管理，这种管理的目的是要让过程能够被共享，复用，并得到持续的改进

联系：管理过程用于管理的实践，软件过程则是具体的实践，管理过程为软件过程做好具体的行动规划。

第三、四单元作业

期中测试

  1. 都说项目经理的标签是：年轻、高薪、持续学习，对此你是如何理解的？
    年轻：项目经理的工作多而复杂，需要统筹和安排各方面的工作的有效进行，需要有较好的精力来完成工作。
    高薪：项目经理既需要对具体的领域的业务有一定的了解，还需要有较好的团队领导力，安排项目计划合理进行的能力。对一个人的能力的要求较高，因此具有较高的回报
    持续学习：正如上面所说，项目经理需要人们有较高和较为全面的能力，而能力本身的学习不可能一步到位，需要在实践中不断总结和学习，为了能适应工作的需求，项目经理需要持续地学习。

  2. 为什么说不能轻易离开团队、不要抱怨问题？请谈谈你的看法。
    就计算机的软件开发而言，现在的软件项目都需要一个团队才能完成，即需要人与人之间合作。在合作中不同的人对项目和任务有不同的理解，我认为这是必然的。当存在分歧时，只有交流才有可能解决问题，如果一味地消极地态度逃避或抱怨，无法达成一致，只会让项目难以进行下去

  3. 如果你是一个项目经理，你如何做好项目的团队管理？

    1. 一个项目经理需要了解一个团队中成员的能力，将合适的任务交给不同的人。
    2. 在项目开发的过程中，需要有能力协调好成员之间的合作顺利进行，有异见时有能力解决
    3. 能在项目进行过程中把控好项目的进度
    4. 当团队中出现过大的分歧时，需要有自己的想法来领导团队

  4. 什么叫做项目的管理性收尾？它有什么重要意义？
    管理性收尾是对开发过程中相关的成果和经验进行总结并存档。
    重要意义：它能帮助我们更好的完成项目的验收与评估，对本次项目的总结有利于我们之后的项目开发过程中经验的借鉴。

  5. 为什么项目中会频繁出现需求方面的错误和问题？我们应该如何去正确地理解需求？
    因为在了解用户的需求交流的过程中，语言的表达往往存在着模糊性、主观性，用户本身对自身需求的了解也存在着不确定性与变化性，因此需求往往会存在着错误和问题。
    如何正确地理解需求：

    1. 识别真正的用户
    2. 正确理解用户需求
    3. 忍耐力与清晰的思维
    4. 说服和教育客户
    5. 建立需求分析小组

  6. 在项目当中，如何科学地进行进度的管理预留？为什么要这么做？
    可采用相关的理论来进行科学的管理预留：

    1. 关键路径法
    2. 时间压缩法
    3. 资源平衡
    4. 关键链法
      这么做的原因：因为在每一项任务的生产过程中，毫无疑问我们会受到夸大任务完成所估算的时间的诱惑。因为我们需要通过管理预留来合理评估任务所需要的时间。因为时间计划的不合理会导致项目无法良好地进行。
