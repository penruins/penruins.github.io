三层架构和MVC设计模型介绍

开发架构一般都是基于两种形式，一种是C/S架构，另一种是B/S架构

在JavaEE 开发中，几乎都是基于B/S架构的开发

B/S 三层架构

  1. 表现层 SpringMVC
  2. Spring
  3. MyBatis

表现层要做的事情

  1. 获取浏览器的请求参数，传递给业务层（请求参数）
  2. 获取业务层的数据，把对应的结果响应给浏览器（响应结果）

MVC模型
Model 模型 （JavaBean的一个对象）
View 视图 （jsp、html）
Controller 控制器 （servlet）

controller receives datas from browser, then 通过JavaBean 把拿到的数据封装起来。
业务层返回的数据，也可以通过JavaBean封装以后再返回给browser

SpringMVC框架的介绍

Spring是一种基于Java的实现MVC设计模型的请求驱动类型的轻量级Web框架，属于Spring Framework的后续产品

SpringMVC已经成为目前最主流的MVC框架之一，并且随着Spring3.0的发布，全面超越Struts2，成为最优秀的MVC框架

它通过一套注解，让一个简单的Java类成为处理请求的控制器，而无需实现任何接口

MVC属于表现层框架

入门案例中使用的组件介绍

SpringMVC框架基于组件方式执行流程

  1. browser->DispatcherServlet Request请求
  2. DispatcherServlet->HandlerMapping 请求查找Handler，让Controller类中方法去执行/hello
  3. HandlerMapping->DispatcherServlet 返回一个执行链，即返回HelloController中sayHello()
  4. DispatcherServlet->HandlerAdapter 请求适配器执行
  5. HandlerAdapter->Handler 执行
  6. Handler->HandlerAdapter 返回ModelAndView,即返回success.jsp
  7. HandlerAdapter->DispatcherServlet 返回ModelAndView
  8. DispatcherServlet->ViewResolver 请求进行视图解析
  9. ViewResolver->DispatcherServlet 返回View
  10. DispatcherServlet->jsp/Freemaker/Excel/pdf 视图渲染，将模型数据填充到request域
  11. DispatcherServlet->browser response响应

在SpringMVC的各个组件中，HandlerMapping, HandlerAdapter, ViewResolver是SpringMVC的三大组件，使用<mvc:annotation-driven>自动加载RequestMappinghHandler和RequestMappingHandlerAdapter

RequestMapping注解的作用

作用：用于建立请求URL和处理请求方法之间的对应关系

RequestMapping注解的属性

  1. name
  2. value
  3. path
  4. method
  5. params
    用于指定限制请求参数的条件。要求请求参数的key和value必须和配置的一模一样
    例如：
    params = {“accountName”} 表示请求参数必须有accountName
    http://localhost:8080/springmvc1/user/testRequestMapping?username=penruins
    params = {“money!=100”} 表示请求参数中money不能是100
    http://localhost:8080/springmvc1/user/testRequestMapping?money=101

path和value是等同的，都是指映射的路径
不写属性名相当于value/path

@RequestMapping(value="/testRequestMapping",method={RequestMethod.POST});

RequestParam注解

作用：把请求中指定名称的参数给控制器中的形参赋值

RequestBody注解

作用：用于获取请求体内容。直接使用得到的是key=value&key=value…结构的数据

PathVariable注解

用于绑定url中的占位符。例如url中/delete/{id}, 这个{id}就是url占位符

HiddentHttpMethodFilter过滤器

作用：由于浏览器form表单只支持GET和POST请求，而DELETE, PUT等method并不支持，Spring3.0 添加了一个过滤器，可以将浏览器请求改为指定的请求方式，发送给我们的控制器方法，使得支持GET, POST,PUT,DELETE请求

RequestHeader注解

作用：用于获取请求消息头
在实际开发中一般不怎么用

CookieValue注解

作用： 用于把指定cookie名称的值传入控制器方法参数

ModelAttribute注解

作用：

  1. 该注解是 SpringMVC4.3 版本以后新加入的。它可以用于修饰方法和参数
  2. 出现在方法上时，表示当前方法会在控制器的方法执行之前，先执行。他可以修饰没有返回值的方法，也可以修饰有具体返回值的方法
    应用场景：我们在编辑一个用户时，用户有一个创建信息字段，该字段是不允许被修改的。在提交表单数据是肯定没有次字段的内容，一旦更新会把该字段内容置为null，此时就可以使用才注解解决问题。
  3. 出现在参数上，获取指定的数据给参数赋值

没有返回值的情况

SessionAttributes注解

作用：用于多次执行控制器方法间的参数共享

拦截器

类似于Servlet开发中的过滤器Filter, 用于对处理器进行预处理和后处理

拦截器链：将拦截器按一定的顺序联结成一条链。在访问被拦截的方法或字段时，拦截器链中的拦截器就会按其之前定义的顺序被调用

过滤器 是servlet规范中的一部分，任何java web工程都可以使用
拦截器 是SpringMVC框架自己的，只有使用了SpringMVC框架的工程才能用。
