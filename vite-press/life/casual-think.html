2026-05-14 ​

  • 现在使用 claude code 配置的模型是 百炼的 coding plan，如果开2个对话，就老是会报 429，只能一个一个执行，好慢呀， 不知道有什么办法可以同时多开，这样效率就又能提升了
    • 我在我的服务器上还有我的工作电脑上2个地方同时进行对话，还是会被429，指定不同的模型也不行，好像没有办法了
  • 使用 plan mode 一方面可以让你理清思路你想要让AI做什么，另一方面是可以规划好一批任务一次提交给AI，让AI可以持续运行把事情做完，效果更高
    • 我发现即使是在 plan mode 下，它也不出完全不会改动你的工程，它还是会编辑你的工程文件
  • 在windows电脑上，我其实更想用 Terminal 来使用 claude code，感觉 Terminal 上的字体显示更舒服，用 IDEA 的话，有 claude code 的插件可以和 claude code cli 进行交互，我可以选中代码直接发送到cli，如果用 Terminal 就不行
  • 其实系统中(一致性要求很高)有同步任务也不是说会有什么太大的问题，只要同步逻辑写对的不出问题
