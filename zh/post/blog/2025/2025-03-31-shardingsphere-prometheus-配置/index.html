    2025-03-31-Shardingsphere-prometheus-配置 #

    把 shardingsphere proxy 搭建好之后，想要把监测数据放到 grafana里面进行监测，但是按照找到的资料来配置好像都不太行，自己研究了一下启动脚本，现在可以正常获取到指标数据了，所以花点时间把这个问题记录一下

    下载了官网提供的 shardingsphere-proxy 的包之后，里面其实已经包含了 agent 相关的 jar 包以及插件相关的 jar 包，其实不需要再下载一个 shardingsphere-agent

    我们需要配置一下agent.yaml里面的内容，文件的位置在 apache-shardingsphere-5.5.2-shardingsphere-proxy-bin/agent/conf/agent.yaml，现在只需要配置 prometheus，所以如下所示

    plugins:
  metrics:
    Prometheus:
      host: "localhost"
      port: 9000
      props:
        jvm-information-collector-enabled: "true"

    在 prometheus 的配置文件中需要加上 9000 的指标接口（如果是使用 brew 安装的话，配置文件在这个位置 /opt/homebrew/etc/prometheus.yml）

    scrape_configs:
  - job_name: 'shardingsphere'
    static_configs:
      - targets: ['localhost:9000']

    网上查到的这篇文章 Apache ShardingSphere Agent 可观察性实用指南 提到需要修改脚本文件，其实是不用修改的，只需要夹一个参数 --agent 就可以

    需要注意的是运行的目录需要在 shardingsphere-proxy 的根目录下，因此命令如下所示

    sh bin/start.sh 5555 --agent
